Root-Me: ELF x86 — Basic

Goal: Analyze the provided ELF binary to find the username/password required to validate the challenge.

1. Identify the file type

Check the file type using the file command:

yoyo@LAPTOP:~/RootMe/cracking$ file ch2.bin
ch2.bin: ELF 32-bit LSB executable, Intel 80386, version 1 (SYSV), statically linked, for GNU/Linux 2.6.8, with debug_info, not stripped

Notes from the output:

  • ELF 32-bit LSB for Intel 80386.
  • The binary is statically linked (all libraries embedded).
  • It contains debug_info and is not stripped, so symbol and debug information are available — making analysis easier.
  • No obfuscation techniques are apparently applied.

2. Run the program

Execute the binary to observe runtime behavior:

dboujnane@krick:~/espaces/travail/M1/S2/sl/rootme/cracking$ ./ch2.bin
############################################################
##        Bienvennue dans ce challenge de cracking        ##
############################################################

username: hello
Bad username

The program prompts for a username (and intuitively also a password).

5. Conclusion / Notes

  • Because the binary contains debug info and is not stripped, analysis is straightforward. strings is sufficient to find the expected credentials in this case.
  • Optional follow-ups: use objdump, readelf or a disassembler (e.g. Ghidra/Radare2/IDA) to inspect control flow or confirm how credentials are compared.
