Root-Me: ELF x86 — No Protection

Category: Binary Exploitation • Difficulty: Easy • Date: 2025-10-23

Goal: Reverse engineer the provided ELF binary to find the password (flag) required to validate the challenge.

1. Identify the file type

Start by checking the file type with file:

yoyo@LAPTOP-9SQVR51F:~/RootMe/cracking$ file ch1.bin
ch1.bin: ELF 32-bit LSB executable, Intel 80386, version 1 (SYSV), dynamically linked, interpreter /lib/ld-linux.so.2, for GNU/Linux 2.6.9, not stripped

Notes from the output:

  • ELF 32-bit LSB for Intel 80386.
  • Binary is dynamically linked and the interpreter is /lib/ld-linux.so.2.
  • The binary is not stripped, so symbol and debug information are available.
  • Given the challenge title and these properties, no obfuscation or protection appears to be in place.

2. Run the program

Execute the binary to observe runtime behavior:

yoyo@krick:~/espaces/travail/M1/S2/sl/rootme/cracking$ ./ch1.bin

The program prompts for a password and prints a failure message when the input is incorrect.
